# Architecture

## 설계 원칙

본 프로젝트는 졸업프로젝트 코드이므로 단순히 동작하는 구현보다 유지보수 가능하고 확장 가능한 구조를 우선한다. 기능 추가 및 리팩토링 시 다음 소프트웨어공학 원칙을 준수한다.

- 단일 책임 원칙을 지킨다.
- UI와 게임 엔진을 분리한다.
- UI는 게임 규칙을 직접 계산하지 않는다.
- 핵심 게임 엔진은 PyQt5에 의존하지 않는다.
- 분석 로직은 가능한 한 별도 모듈로 분리한다.
- ZiNi 계산은 MinesweeperEngine 내부에 직접 넣지 않고 별도 계산기로 구현한다.
- 기능 추가 시 기존 동작을 깨지 않도록 단계별로 리팩토링하고 테스트한다.
- 확장성, 유지보수성, 테스트 가능성을 고려하여 코드를 작성한다.

## 현재 파일 구조

- `main.py`: 프로그램 진입점
- `core_engine.py`: 지뢰찾기 핵심 게임 엔진
- `ui_manager.py`: PyQt5 기반 UI
- `requirements.txt`: 실행에 필요한 라이브러리 목록
- `.gitignore`: Git 추적에서 제외할 파일 목록

## ZiNi 구현 방향

ZiNi는 정확한 이론적 최소 클릭 수가 아니라, 특정 ZiNi 알고리즘이 계산한 추정 최소 클릭 수로 다룬다.

1차 구현 대상은 `G.ZiNi`이다.

`H.ZiNi`와 `8-way G.ZiNi`는 후속 확장 후보로 둔다.
